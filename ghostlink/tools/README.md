# GhostLink Tools

The MAX kernel exposes the following tool primitives:

- `MAP` — action `parse` via multipaths PLN-1a-skeleton, PLN-1b-lex, PLN-1c-ast, PLN-1d-normalize, PLN-1e-index.
- `CLEANSE` — action `scrub` via multipaths PLN-2a-trim, PLN-2b-dedup, PLN-2c-noise, PLN-2d-validate, PLN-2e-sanitize.
- `SURGE` — action `accelerate` via multipaths PLN-3a-fastscan, PLN-3b-batch, PLN-3c-parallel, PLN-3d-throttle, PLN-3e-postcheck.
- `REFLECT` — action `mirror` via multipaths PLN-6a-snapshot, PLN-6b-compare, PLN-6c-delta, PLN-6d-verify, PLN-6e-report.
- `RECAST` — primitive without a dedicated pipeline entry in the MAX seed.
- `RETURN` — primitive without a dedicated pipeline entry in the MAX seed.
- `WRENCH` — primitive without a dedicated pipeline entry in the MAX seed.
- `RECALL` — primitive without a dedicated pipeline entry in the MAX seed.
- `LOCK` — action `bound` via multipaths PLN-4a-caps, PLN-4b-scope, PLN-4c-roles, PLN-4d-ratelimit, PLN-4e-freeze.
- `SILENCE` — action `mute` via multipaths PLN-5a-output, PLN-5b-logs, PLN-5c-events, PLN-5d-network, PLN-5e-hardware.
- `SHELL` — primitive without a dedicated pipeline entry in the MAX seed.
- `MACROS` — primitive without a dedicated pipeline entry in the MAX seed.
- `MIRROR` — primitive without a dedicated pipeline entry in the MAX seed.
- `SHADOW` — primitive without a dedicated pipeline entry in the MAX seed.
- `BIND` — action `fuse` via multipaths PLN-9a-join, PLN-9b-conflict, PLN-9c-weights, PLN-9d-resolve, PLN-9e-commit.
